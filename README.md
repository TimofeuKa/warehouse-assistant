## Стек

- Backend: ASP.NET Core Web API, .NET 10
- ORM: Entity Framework Core 10
- DB: PostgreSQL
- Frontend: Vue 3 + Vite + TypeScript
- Tests: xUnit
- API UI: Scalar + OpenAPI

## Реализовано

- Справочник складов с полями 'Id' и 'Name'.
- Начальное заполнение справочника 3 складами.
- Справочник номенклатур с полями 'Id' и 'Name'.
- Начальное заполнение справочника 7 номенклатурами.
- Приход товара извне на выбранный склад.
- Расход товара со склада за пределы компании.
- Перемещение товара между двумя складами компании.
- Добавление нескольких ТМЦ в одно движение.
- Запрет повторения одной номенклатуры внутри движения.
- Учет количества в целых положительных единицах.
- Журнал всех движений с датой, направлением и составом ТМЦ.
- Удаление движения с последующим пересчетом остатков.
- Отчет по остаткам на выбранном складе на указанный момент времени.
- Пользовательский интерфейс для создания прихода, расхода и перемещения.
- Выбор склада и времени формирования отчета по остаткам.

## Unit-тесты

Бизнес-логика покрыта 11 тестами на xUnit:

1. Расчет остатков после прихода, расхода и перемещения.
2. Исключение будущих движений из отчета на выбранное время.
3. Запрет повторения номенклатуры в одном движении.
4. Запрет нулевого количества.
5. Обработка 'items: null' без внутренней ошибки сервера.
6. Обработка пустого списка 'items: []'.
7. Запрет одинакового склада отправления и назначения.
8. Запрет расхода, который создает отрицательный остаток.
9. Запрет операции задним числом, создающей отрицательный остаток в будущем.
10. Запрет удаления прихода, используемого последующим расходом.
11. Разрешение безопасного удаления расхода с корректным пересчетом остатка.

## Реализовано сверх ТЗ

- Запрет отрицательных остатков при расходе и перемещении.
- Проверка всей временной последовательности при создании операции задним числом.
- Запрет удаления движения, если удаление нарушит последующие остатки.
- Проверка существования складов и номенклатур до сохранения движения.
- Проверка 'null', пустого списка ТМЦ и неположительного количества.
- Дублирование критичного правила уникальности номенклатуры индексом PostgreSQL.
- Отдельные DTO для входных и выходных данных API.
- Разделение Minimal API endpoints по отдельным файлам.
- Docker Compose для PostgreSQL с постоянным volume и healthcheck.
- CORS-политика для локального Vue-приложения.
- Подтверждение перед удалением движения.
- Загрузочные, пустые и ошибочные состояния интерфейса.
- Явный выбор типа операции: приход, расход или перемещение.
- Безопасная миграция количества с 'numeric(18,3)' на 'integer' без округления дробных данных.
